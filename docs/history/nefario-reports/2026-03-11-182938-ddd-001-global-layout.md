## Summary

Created DDD-001-global-layout.md, the foundational layout contract for the Mostly Hallucinations blog. Defines a two-tier width model (--measure 68ch reading column within --layout-max 1200px), three-tier spacing scale, responsive breakpoint padding, and CSS architecture for integrating tokens.css with the AEM Edge Delivery Services boilerplate.

## Original Prompt

A Design Decision Document (DDD-001-global-layout.md) exists at docs/design-decisions/ that defines the page chrome — max content width, margins, responsive breakpoints, and section spacing — so that all subsequent design surfaces (header, footer, post index, post detail) build on an approved foundation.

## Key Design Decisions

1. **Two-tier width model**: --layout-max (1200px) constrains all section containers; --measure (68ch) constrains reading prose in .default-content-wrapper. Blocks can use the full layout-max width.

2. **Token loading via separate `<link>`**: tokens.css loads as a separate stylesheet in head.html before styles.css, avoiding @import render-blocking chains.

3. **Full boilerplate replacement**: All boilerplate :root variables in styles.css are replaced wholesale with token names — no mapping layer or indirection.

4. **Three-tier spacing scale**: --space-paragraph (1em), --space-element (1.5em), --section-spacing (48px). Em units tie rhythm to responsive font sizes.

5. **Code block escape hatch**: .default-content-wrapper must not use overflow:hidden, allowing code blocks to extend beyond --measure.
