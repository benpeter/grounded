## Original Prompt

A Design Decision Document (DDD-003-footer.md) exists at docs/design-decisions/ that defines the footer as a single understated line — copyright, LinkedIn link, Legal Notice, Privacy Policy separated by middots — so that the existing footer block at blocks/footer/ can be redesigned to match the brand's minimal aesthetic.

## Key Design Decisions

1. **Single `<p>` with inline links**: All footer content in one paragraph element with inline links and middot text nodes. This avoids EDS `decorateButtons()` converting lone links into pill-shaped buttons — a technical constraint, not a style preference.

2. **Background: `--color-background` (not `--color-background-soft`)**: Current boilerplate uses a tinted band. The warm-white aesthetic requires the footer to be invisible as a surface — the top border provides structural separation instead.

3. **Links underlined by default**: Unlike the header (which is entirely one link), the footer has inline links among non-link text. `text-decoration: underline` provides WCAG 1.4.1 compliance via non-color indicator, since `--color-link` vs `--color-text-muted` achieves only ~1.34:1 (below the 3:1 threshold for color-only distinguishability).

4. **Responsive alignment shift**: Centered on mobile (< 600px) for visual balance when text wraps to two lines; left-aligned at tablet+ (>= 600px) to match header and body text alignment.

5. **Symmetric bookends with header**: Footer mirrors header's border treatment (`1px solid var(--color-border-subtle)` at top) and uses the same `--content-padding-*` tokens and `--layout-max` constraint.

6. **No JS changes needed**: Current boilerplate `footer.js` (load fragment, clear, append) is sufficient. CSS alone reshapes the output.
