## Key Design Decisions

1. **Type badge placement**: Stacked above title on its own line (not inline). Better mobile wrap behavior at 375px.
2. **Heading level**: `<h2>` for post titles directly under visually hidden `<h1>`. No intermediate section headings on the home page.
3. **Type badge accessibility**: Dual pattern — visible badge `aria-hidden="true"` + sr-only prefix inside `<h2>` for heading navigation.
4. **All four types identical visual treatment**: The type label text provides sufficient differentiation. Site aesthetic rules prohibit per-type decorative styling.
5. **Tags as `<ul>` styled inline**: Semantic list for screen readers, CSS displays inline with middot separators.
6. **Title uses `--color-heading`**: Not `--color-link`. Title reads as a heading; link affordance via hover underline.
7. **Tags as links from day one**: Will 404 until DDD-007 ships. Avoids re-implementing tag rendering.
8. **No `role="feed"`**: Implies infinite scroll, which this site does not use.
9. **`helix-query.yaml` as hard prerequisite**: Elevated from Open Question to Context section dependency per user direction.
