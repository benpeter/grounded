## Security Review: DDD-004 Home Post Index

**Verdict: ADVISE**

### Summary

The plan is security-conscious. The mandatory `createElement`/`textContent`/`setAttribute` pattern eliminates XSS from all data fields. Tag slug validation with `/^[a-z0-9-]+$/` blocks path injection in tag hrefs. No secrets, no auth, no cross-origin fetch, no eval. One gap warrants a low-severity advisory.

---

### Finding

#### LOW — Missing href validation on `path` field from query-index.json

**Location**: Task 2, `post-index.js` — the `<a href="{path}">` in the title link

**Description**: The `path` field from `query-index.json` is used directly as the title link `href` without validation. The plan validates tag slugs but does not include the same treatment for `path`.

**Impact**: A `javascript:` scheme value in `path` would create a javascript: link. The actual risk is low — `query-index.json` is generated by the EDS platform from CMS-authored page paths, not user input. No external write path exists. Modern browsers also impose some mitigations. But it is an inconsistency given that tag slugs are explicitly validated.

**Remediation**: In `post-index.js`, validate `path` starts with `/` before using as `href`:

```js
if (!entry.path || !entry.path.startsWith('/')) continue; // skip entry with invalid path
```

Or at minimum, reject javascript: and data: schemes:

```js
const safePath = /^\//.test(entry.path) ? entry.path : null;
if (!safePath) continue;
```

This is consistent with the tag slug validation philosophy already in the plan and closes the field completely.

---

### Confirmed Secure

- **XSS**: Explicit `createElement`/`textContent`/`setAttribute` mandate with no `innerHTML` — correct
- **Tag hrefs**: `/^[a-z0-9-]+$/` validation before use in href — correct
- **Fetch target**: Relative URL `/query-index.json`, same-origin only — no SSRF surface
- **CSP**: No inline handlers, no eval, no dynamic script injection — compliant
- **Secrets**: No credentials, API keys, or sensitive data in JS, CSS, or YAML — clean
- **Supply chain**: No external dependencies added — no new attack surface

### Recommendation

Add path validation to Task 2's prompt. One line of code. The implementation agent should treat it as on par with the tag slug validation that is already explicitly required.
